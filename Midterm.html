The dependent variable is "spam" where one indicates that an email is spam and zero otherwise. Which three variables in the dataset do you think will be important predictors in a model of spam? Why?

One of the important features could be total capital run length, because having long strings of capitals is not usually an accepted semantic way to write non-spam emails. The more capital run length could be thus correlated with being a spam email. Because capital run length average is correlated with the longest capital run length and the total of capital run length, I will only choose the total to represent that feature to avoid multicollinearity of coefficients.

The next feature that could be important is the word frequency of order, because often times spam emails could be trying to get consumers to buy things or click on c ertain links. The last feature I will choose is character frequency, since spam messages usually have interrupted patterns with characters, and I will arbritarily choose the character frequency of the semicolon to see if it is highly correlated with the presence of a spam email.

The main reason I'm choosing these three is because it is likely that character frequency of one type of character is correlated with another, and that word frequency of one word is correlated with another, so again, to avoid correlation of predictors to have clean predictive power, I will choose the three most independent features.

5. Describe the importance of training and test data. Why do we separate data into these subsets?¶

When we first have a data set, we split it into training and test data because we want to use a training set to build and tune a model, and a test set to estimate the predictive performance of our original model. Usually we randomly split the training and test data, to have an effect of random sampling of data points, so that the test data simulates potentional data that our model would use to make new predictions for y. We then compare with our test data how well our model predicted each y. This helps validate our model, built on the training data, to the real world.

Essentially the subsets allow us to build a model and then test how well that model works before we apply the model into places we do not have both an x and a y. Randomly splitting the data using sampling techniques such as stratification and k-fold will also help keep the training and test data homogenous so that our observations remain under the assumption of being independent and identically distributed.

6. What is k-fold cross validation and what do we use it for?¶

K-fold cross validation is a resampling technique that evaluates generalization of performance. It is more stable than just doing a simple train/test split of the data. It splits the data multiple times into k partitions, called "folds". Then, many models are trained: the first model on the first fold is the test set, and the other folds are used for training. Then the accuracy of each of the folds except for the first is compared to fold 1. Then, this process is repeated by now using fold 2 as a test set and all the others are training. There will be a total of k accuracy values. Train-test-splitting only performs one random split of the data, while CV performs multiple splits, which shows how well the model generalizes to all samples of a dataset. It provides both the best and worst case scenarios when applied to the data, thus helping the data be used more effectively. The cross_val_score function builds these multiple models internally, and cv evaluates how well the model generalizes. As k increases, the bias of the model becomes smaller. Thus this provides a robust metric as to why we would choose one model over another.

7. How is k-fold cross validation different from stratified k-fold cross validation?¶

Stratified k-fold splits the data into k folds but starts with the first one-kth-part of the data. This creates balance with respect to outcome. Essentially, it splits such that the proportions between classes in the data are the same in each fold as they are in the whole dataset. This promotes homogeneity of folds. For example, if 90% of samples are class A and 10% class B, then 90% of each sample will be class A and 10% class B. This homogeneity of folds, due to the ensurance of relative class frequencies with respect to the full sample, makes stratified k-fold more reliable for generalization.

12. Now rerun your best model from questions 8 through 11, but this time add three new variables to the model that you think will increase prediction accuracy. Did this model predict test data better than your previous models?¶

Note I changed the tuning paramters to max_depth= 11 and criterion= entropy because grid search cv changed its best parameters due to adding new variables. This makes sense considering more data points mean one can have a deeper tree without increasing model complexity too much. Accuracy is around the same, with about 79% accuracy with kfold cv, as it was before adding the variables. However, we only added 3 variables, and in general, if we added all or more of them, adding new ones would increase prediction in the long run until it hit a peak at which point adding new variables would create an overfitting problem. Adding new variables at this stage though when we only were using 3 would increase variance, and thus, due to the variance-bias trade off, decrease bias. Decreasing bias means we are moving closer to the true parameters and increasing our prediction power overall. This model does the same as with three variables under the random forest, but much better than all other models overall.¶
